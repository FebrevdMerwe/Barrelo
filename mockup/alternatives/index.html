Same problem, five layouts

The board+scorecards eat too much space above the game area. Each alternative below solves that a different way — pick one to build, not to admire.

Previously today

Fixed split

Board + scorecards always on top, game panel always on the bottom. Simple, but the board is "on" even when nobody's about to throw. Superseded — this reference now only lives here, since match.html was rebuilt.

Baseline for comparison (retired) Alt A

Collapsed input rail

Board+scorecards shrink to a slim strip at rest and only expand to full size while a dart is being entered. Single screen, no new architecture.

Space win vs. a bit more interaction (tap to expand) Alt B

Scorer + spectator split

Two screens: a scorer device with only the board/input, and a spectator/TV page with only the game view, full-bleed. Matches the mobile-companion / TV-display use case already in the plan.

Biggest space win, but needs a second device + live sync Alt C

Injectable widgets

Dartboard and player-round become self-contained, reusable widgets (events in, state out) that each game composes into its own layout — same look, different position.

Max layout freedom vs. every plugin now owns composition
Alt D · mine, now built

On-demand input drawer

Game view owns the whole screen, always. The board lives in a bottom sheet that's a thin "your turn" handle by default and only fully opens mid-throw. This is the one that got picked.
